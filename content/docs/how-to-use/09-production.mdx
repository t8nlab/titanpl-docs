Titan production builds generate highly-optimized bundles designed to run on the **Titan Engine**.  
You do not need to compile any Rust code yourself.

JavaScript is executed via an embedded **V8 runtime** within the engine, while **all networking, concurrency, and async execution are handled by Rust**.

---

## 🏗️ Production Build

From your Titan project root, run:

```bash
npm run build
````

<Steps>
  <Step title="Action Discovery">
    Titan locates your route definitions and actions in `app/`.
  </Step>

<Step title="Bundle actions">
All JavaScript/TypeScript actions are aggressively optimized, tree-shaken, and bundled by `@titanpl/packet`.
</Step>

<Step title="Emit to dist">
The finalized artifacts are emitted to the `dist/` directory, ready to be consumed by the Engine.
</Step> </Steps>

On success, you will see output similar to:

```text
★ Titan Packet: Building production output...
→ Bundling actions...
→ Emitting to dist/...
✔ Titan production build complete!
```

## 🚀 Deployment

<Callout type="info">
  Deploying a Titan application is as simple as deploying your `dist/` folder and `package.json`.
</Callout>

### Typical deployment flow

1. Run `npm run build` locally or in CI
2. Upload the `dist/` folder and `package.json` to your server
3. Run `npm install --omit=dev` (installs the engine binary natively)
4. Start the server using `npm run start`

That’s it.

<Callout type="info">
  **Production Tip:** To ensure cross-platform stability, you can manually add the engine packages to your `optionalDependencies` in `package.json`. This ensures the correct binary is available for your production environment.
  
  ```json
  "optionalDependencies": {
    "@titanpl/engine-linux-x64": "2.0.5",
    "@titanpl/engine-win32-x64": "2.0.3"
  }
  ```
</Callout>

Works on:

* Any Node.js-capable VPS
* Bare metal
* Docker / Cloud Containers
* Any platform supporting native binaries

## ▶️ Running in Production

Run the engine:

```bash
npm run start
```

Titan instantly boots the pre-compiled Titan Engine pointing to your `dist/` folder, primed for maximum throughput.

## ⚡ JavaScript Runtime in Production (V8)

<Callout type="info">
  Gravity does <strong>not</strong> limit your backend performance.
</Callout>

Facts:

* JavaScript runs via **V8**, embedded securely in the Rust Engine
* **HTTP, async I/O, threading, and scheduling are fully Rust-owned**
* Actions execute synchronously across a multi-threaded isolate pool

V8 is used **only for business logic**.
Rust owns the performance-critical hot path.

Result:

* Predictable latency
* Stable throughput
* No dev vs prod behavior gap

## ☁️ Deploy on Railway / Render (GitHub → Production)

<Callout type="info">
  For Railway and Render, you do <strong>not</strong> deploy the binary manually.  
  Just push your Titan project to GitHub and connect the repository.
  Don't forget to add the `.gitignore` file (refer to [Environment Variables](/docs/how-to-use/06-env) for details).
</Callout>

Titan projects ship with a **fully production-ready Dockerfile**.  
Railway and Render automatically detect it and handle the entire build and runtime process.

### How it works

1. Push your full Titan project to GitHub  
2. Connect the repository to **Railway** or **Render**
3. Click deploy

That’s it.

The platform will:

- Run `npm run build`
- Start the server using `npm run start`

You do **not** need to:

- Hand-configure Rust
- Write a custom build pipeline
- Manage complex process watchers

### What gets deployed

- Your bundled `dist/` output
- The pre-compiled `@titanpl/engine-<os>-<arch>` natively matched to the host container.

The provided Dockerfile handles **everything** required for production.

### Recommended settings

- **Service type:** Docker
- **Build source:** GitHub repository

Railway and Render both work out of the box.

---

**Summary**

- Push project to GitHub
- Connect repo to Railway or Render
- Dockerfile builds and runs Titan automatically
- Zero manual production configuration
